# Frontend UI Spec to React

用于把业务需求快速转换为可实施的前端页面结构。

## Workflow

1. 明确输入约束。
收集页面目标、用户角色、数据来源、交互状态和响应式要求。

2. 拆分信息架构。
把页面拆成区块、组件和复用单元，先确定层级再写实现细节。

3. 输出实现草案。
给出组件树、关键 props、状态流向和最小可运行页面骨架。

4. 校验可实现性。
检查可访问性、空态/异常态、移动端适配和可维护性。

## Output Format

默认按以下结构输出：

- 页面目标与约束
- 组件树与职责
- 数据与状态流
- 实现建议（文件结构 + 关键代码片段）
- 风险与后续优化项

## Rules

- 不直接堆砌视觉描述，优先输出可编码的信息。
- 组件命名保持清晰，遵守单一职责。
- 对不明确需求先列假设，再继续生成方案。
