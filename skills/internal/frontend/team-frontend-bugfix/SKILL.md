# Frontend Bugfix

聚焦前端问题排查与回归防护。

## Workflow

1. 复现问题。
明确环境、步骤、期望结果与实际结果，优先构造最小复现路径。

2. 定位根因。
结合控制台日志、网络请求、状态变化和组件生命周期判断故障点。

3. 最小修复。
在不扩大改动面的前提下修复根因，避免引入二次回归。

4. 回归验证。
覆盖核心路径、边界条件和历史高频问题场景。

## Output Format

- 问题描述与复现步骤
- 根因分析
- 修复方案与变更点
- 验证结果
- 后续防护建议（测试或监控）

## Rules

- 没有复现前不下结论。
- 修复必须解释“为什么有效”。
- 优先补充最小必要测试，防止同类问题重复出现。
