# Backend API Contract Review

用于后端接口契约的结构化评审。

## Workflow

1. 收集上下文。
明确接口用途、调用方、版本策略和发布窗口。

2. 审核契约一致性。
检查字段命名、类型约束、必填规则、状态码语义和错误码规范。

3. 评估兼容性。
识别破坏性变更，给出灰度策略、版本迁移或兼容兜底方案。

4. 形成评审结论。
按严重级别输出问题清单和上线建议。

## Output Format

- 评审范围
- 高风险问题（必须修复）
- 中低风险问题（建议修复）
- 兼容性结论
- 上线前检查清单

## Rules

- 兼容性问题优先级最高。
- 评审意见必须可执行，避免抽象描述。
- 对每个风险点明确影响调用方与修复建议。
